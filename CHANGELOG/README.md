# CHANGELOG

מטרה: מעקב ברור אחרי קבוצות שינויים בפרויקט, כולל הסיבה לשינוי וההשפעה שלו.

## איך עובדים כאן

1. כל קבוצת שינויים נכנסת לקובץ אחד.
2. כל קובץ חייב להסביר גם **מה השתנה** וגם **למה השתנה**.
3. כל קובץ חייב לכלול הפניה לקומיט/ים רלוונטיים.

## חלוקה לקבוצות

- `dashboard/` שינויי Dashboards/UI/Reports
- `docs/` שינויי תיעוד
- `tools/` שינויי סקריפטים וכלי עזר
- `firmware/` שינויי Zephyr/embedded/app logic
- `infra/` שינויי CI/CD, מבנה ריפו, תשתיות

אם שינוי שייך ליותר מקבוצה אחת:
- יוצרים את הרשומה בקבוצה המרכזית
- מוסיפים הפניה ב-`INDEX.md` לקבוצות המשניות

## Naming Convention

שם קובץ:

`YYYY-MM-DD-short-topic.md`

דוגמה:

`2026-03-05-remove-legacy-phases-dashboard.md`

## שדות חובה בכל רשומה

- `Context` מה הטריגר/הצורך שהוביל לשינוי
- `Changes` מה שונה בפועל
- `Why` למה קיבלנו את ההחלטה
- `Impact` השפעה למשתמשים/מפתחים
- `References` קומיטים וקבצים מרכזיים
